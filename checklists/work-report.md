# Work Report Template

Use this template for every completion report. Never report "Done" without filling this out.
Omitting sections is not allowed — write "N/A" with a reason if a section does not apply.

---

## What was built / changed / removed

[Describe specifically what was done. Not "I implemented the feature" but what files were created,
what functions were written, what behavior now exists that did not before.]

## What was NOT built (scope explicitly excluded)

[List anything from the original request that was not implemented, and why.
"Not applicable" is only valid if the original request was fully simple and complete.]

## Assumptions made

[List every assumption made during this task. If none: state "No assumptions made."]
- Assumed: [what] because [why]
- Confirmed by user: [yes/no]

## Edge cases not handled

[List every edge case you are aware of that is not handled by the current implementation.]

## Tests

- Written for: [list what is tested]
- Not written for: [list what is not tested and why]
- Existing tests: passing / failing / not checked

## Known issues

[Anything you found during this task that is not part of the task scope but should be addressed.
These are observations only — do not fix them without instruction.]

## What to do next (if applicable)

[If the task is partial, state clearly what the next step is and what state the code is in.]

---

A completion report that says "Done!" with none of the above is not a completion report.
It is a guess that everything is fine. Guesses rot codebases.
